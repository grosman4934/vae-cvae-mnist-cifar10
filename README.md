# VAE и Conditional VAE для MNIST и CIFAR-10

Лабораторная работа по реализации вариационного автоэнкодера (VAE) и условного вариационного автоэнкодера (Conditional VAE, CVAE) для реконструкции и генерации изображений на датасетах MNIST и CIFAR-10.

## Описание проекта

В проекте рассматриваются генеративные модели на основе вариационных автоэнкодеров.  
Были реализованы модели, которые кодируют изображение в латентное пространство, восстанавливают его через декодер и позволяют генерировать новые изображения из случайного шума.

Работа выполнена в Jupyter Notebook с использованием TensorFlow/Keras.

## Что реализовано

- вариационный автоэнкодер для MNIST;
- условный вариационный автоэнкодер для MNIST;
- вариационный автоэнкодер для CIFAR-10;
- условный вариационный автоэнкодер для CIFAR-10;
- сверточные энкодеры и декодеры;
- reparameterization trick для выборки из латентного распределения;
- функция потерь на основе вариационной нижней границы;
- KL-дивергенция между апостериорным распределением и стандартным нормальным prior;
- генерация изображений из случайных латентных векторов;
- условная генерация изображений по заданной метке класса;
- EarlyStopping для предотвращения переобучения.

## Используемые датасеты

### MNIST

MNIST содержит изображения рукописных цифр размером `28×28`.  
Для этого датасета была реализована модель с латентным пространством размерности `10`.

Для MNIST использовалась функция потерь, включающая:

- бинарную кросс-энтропию для ошибки реконструкции;
- KL-дивергенцию для регуляризации латентного пространства.

### CIFAR-10

CIFAR-10 содержит цветные изображения размером `32×32×3`, относящиеся к 10 классам:

- airplane;
- automobile;
- bird;
- cat;
- deer;
- dog;
- frog;
- horse;
- ship;
- truck.

Для CIFAR-10 была использована более глубокая сверточная архитектура и латентное пространство размерности `30`.

## Архитектура моделей

### VAE

Обычный VAE состоит из трех основных частей:

1. **Encoder**  
   Преобразует изображение в параметры латентного распределения:
   - среднее значение `t_mean`;
   - логарифм дисперсии `t_log_var`.

2. **Sampling layer**  
   Выполняет выборку из распределения с помощью reparameterization trick:

   ```python
   t = mean + std * epsilon
   ```

3. **Decoder**  
   Восстанавливает изображение из латентного вектора.

### Conditional VAE

В Conditional VAE к входным данным добавляется информация о классе изображения.

Для этого:

- метки классов кодируются one-hot векторами;
- метка подается в энкодер вместе с изображением;
- метка также объединяется с латентным вектором перед декодером;
- при генерации можно задавать нужный класс изображения.

## Основные результаты

### MNIST

Для MNIST модель показывает хорошие результаты:

- реконструированные цифры остаются узнаваемыми;
- Conditional VAE позволяет генерировать цифры заданного класса;
- изображения после генерации имеют характерное для VAE небольшое размытие, но сохраняют структуру цифр.

### CIFAR-10

Для CIFAR-10 задача оказалась значительно сложнее:

- реконструкции получились размытыми;
- сгенерированные изображения плохо различимы;
- условная генерация по меткам работает, но качество изображений остается низким.

Это связано с тем, что CIFAR-10 содержит цветные изображения с большим количеством деталей, фонов и внутриклассовой вариативности.

## Что было сделано для улучшения качества

В ходе работы были предприняты следующие улучшения:

1. Добавлена ранняя остановка обучения с восстановлением лучших весов.
2. Использованы сверточные слои в энкодере и декодере.
3. Варьировалась размерность латентного пространства.
4. Были проведены эксперименты с коэффициентом регуляризации при KL-дивергенции.
5. Для CIFAR-10 была использована более глубокая сверточная архитектура.

## Вывод

Вариационный автоэнкодер хорошо подходит для простых изображений, таких как MNIST.  
Модель успешно обучается реконструировать цифры и генерировать новые примеры.

Для CIFAR-10 базовой VAE/CVAE-архитектуры оказалось недостаточно: изображения имеют более сложную структуру, поэтому качество реконструкции и генерации ниже. Для улучшения результата можно использовать более мощные сверточные архитектуры, увеличить время обучения, подобрать beta-параметр для KL-дивергенции или рассмотреть более современные генеративные модели.
